    Introduction

    Welcome to the Postit v1 API! You can use our API to access Postit API endpoints, with which you can create and access user and group data in our database.

    Postit API is organized around REST and uses HTTP verbs and response codes that most clients are familiar with. All requests must be formatted as Content-Type: application/json (except where noted). All responses are also formatted JSON, encoded in the UTF-8 character encoding (Content-Type: application/json; charset=utf-8). The Accept header will be ignored for all requests.

    Base API Endpoint

    https://postit-ray.herokuapp.com/api/v1

    The API is implemented in Javascript(Nodejs). You can view code examples in the dark area to the right.

    Authentication

    Some endpoints (routes) in this API are protected. They require an access token (x-auth) in the request headers. You can get this token when you signup as a new user or login as existing user.

    To authorize requests to protected endpoints, set the x-auth token in the request header like this::

    {
  "x-auth": "YOUR_AUTH_TOKEN"
}
    You must replace YOUR_AUTH_TOKEN with your own x-auth token, which the server sends back in the response headers when you sign up or sign in.

    Users

    To use the API, you would have to signup as a user. Users can create groups, search other users and add them to their groups, leave groups they have beeen added to, delete groups they have created, and send messages in groups to which they belong.

    Some routes here are protected. You would need to set the x-auth token in the request header to access them

    Signup new user

    This endpoint allows new users create accounts and access protected routes. If the request is successful, the response body would contain a message that welcomes the new user, and a user object that contains user data. This user object contains:

    Request Parameters

    Parameter Description
    username required
    email required
    password required

    Response Parameters

    Parameter Description
    id A unique API generated integer that identifies the user resource on the database.
    username The username used to sign up.
    email The email used to sign up
    about Defaults to New to Postit. This can be changed later.
    imageUrl Defaults to an empty string.
    createdAt Time the user record was created in the database.
    updatedAt Last time the user record was modified.

    Sample Request:

    {
  "username": "daenerys"
  "email": "theunburnt@asoiaf.com"
  "password": "dracarys" 
}

    Sample Success Response:

    {
  "message": "welcome daenerys",
  "user": {
    "id": 5,
    "username": "daenerys",
    "email": "theunburnt@asoiaf.com",
    "about": "New to Postit",
    "imageUrl": "",
    "createdAt": "2017-11-16T08:45:49.906Z",
    "udatedAt": "2017-11-16T08:45:49.906Z",
  }
}

    Sample Error Responses:

    • No username provided (status 400)

    {
  "error": "Email is required"

  "error": "Username cannot be longer than 12 characters"

    Groups

    Groups are communication channels that provide a platform for sharing messages amongst members. Groups can be either public or private. Currently, there is no difference in implementation. Expect huge updates soon.

    Besides the endpoint for creating new groups, other group endpoints require a groupid as a url parameter

    All group routes are protected, and so require authentication. You must set the x-auth token in the header to access them

    Create new group

    A registered user can create a new group with this endpoint.

    description not required

    Errors

    This is a list of the status codes you might see when you encounter an error

    The Postit API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- You did not quite structure your request as you should have.
    401 Unauthorized -- Your x-auth access token is invalid.
    403 Forbidden -- You cannot access this routes, because you have insufficient permissions although authenticated.
    404 Not Found -- The specified group or user, or even route, could not be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
